# Data Model: Avaliacao do RAG

## EvaluationDataset

Versioned input file for evaluation runs.

Fields:

- `dataset_version`: Stable version string for the dataset.
- `description`: Optional human-readable purpose.
- `defaults`: Optional default evaluation settings.
- `cases`: Non-empty list of `EvaluationCase`.

Validation:

- `dataset_version` is required.
- Case ids must be unique.
- Dataset must include at least one unanswered case for acceptance.

## EvaluationCase

One question and its expected behavior.

Fields:

- `id`: Stable case id.
- `question`: Query text.
- `kind`: One of `known_answer`, `unanswered`, `exact_term`, `semantic`.
- `limit`: Optional override for retrieval limit.
- `min_score`: Optional override for search threshold.
- `filters`: Optional category/tag/project filters.
- `expected_chunks`: Expected `ExpectedChunkRef` list for retrieval metrics.
- `expected_category`: Optional category expectation.
- `expected_project`: Optional project expectation.
- `expected_answer_points`: Essential phrases or facts for answer correctness.
- `expected_refusal`: Whether the answer should refuse due to missing context.
- `tags`: Optional labels for slicing reports.

Validation:

- `known_answer`, `exact_term` and `semantic` cases need at least one expected chunk or expected answer point.
- `unanswered` cases must set `expected_refusal` or use `kind = unanswered`.
- Expected answer checks use normalized, case-insensitive text matching in MVP.

## ExpectedChunkRef

Stable reference to an expected retrieval target.

Fields:

- `source_public_id`: Preferred source UUID/public id.
- `source_title`: Optional title fallback for review.
- `chunk_index`: Optional chunk index.
- `page`: Optional page.
- `section`: Optional section.
- `content_hash`: Optional hash for exact fixture matching.
- `snippet`: Optional short content clue for humans.

Validation:

- At least one stable locator must be present.
- Committed examples must avoid sensitive personal data.

## EvaluationRunConfig

Configuration captured at runtime for reproducibility.

Fields:

- `dataset_version`
- `dataset_hash`
- `git_revision`
- `retrieval_limit`
- `min_score`
- `embedding_provider`
- `embedding_model`
- `embedding_version`
- `embedding_config_hash`
- `llm_provider`
- `llm_model`
- `mode`: `baseline`, `candidate` or `search_only`

## EvaluationCaseResult

Per-case output.

Fields:

- `case_id`
- `retrieved_chunks`
- `answer`
- `citations`
- `recall_at_k`
- `reciprocal_rank`
- `answer_correct`
- `refusal_correct`
- `citations_correct`
- `supported_by_sources`
- `latency_ms`: embedding/search/answer/total
- `errors`

## EvaluationReport

Machine-readable report generated by the runner.

Fields:

- `report_version`
- `created_at`
- `run_config`
- `case_results`
- `metrics`
- `thresholds`
- `decision`: `passed`, `failed` or `inconclusive`
- `failures`

## ThresholdConfig

Minimum quality and maximum latency gates.

Fields:

- `min_recall_at_k`
- `min_mrr`
- `min_answer_correct_rate`
- `min_refusal_correct_rate`
- `min_citation_correct_rate`
- `max_embedding_latency_p95_ms`
- `max_search_latency_p95_ms`
- `max_answer_latency_p95_ms`
